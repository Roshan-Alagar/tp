# Roshan Premil - Project Portfolio Page

## Project Overview

**RLAD (Record Losses And Debt)** is a minimalist Command-Line Interface (CLI) finance tracker built for power users who prefer speed and simplicity over GUI applications. Users can record, filter, sort, search, modify, and summarize financial transactions entirely from the terminal.

### Features Implemented

#### 1. `DeleteCommand` (Section 4.2)
Removes a transaction permanently by its HashID.

- Parses `--hashID` from raw arguments
- Validates the flag is present and non-empty before execution
- Looks up the transaction via `TransactionManager.findTransaction()`
- Removes from both the `ArrayList` and `HashMap` in `TransactionManager`
- Shows a success message with the deleted transaction's details, or throws `RLADException` if not found

**Command format:** `delete --hashID a7b2c3`

---

#### 2. `ModifyCommand` (Section 4.3)
Updates specific fields of an existing transaction without deleting and re-adding it.

- Accepts any combination of `--type`, `--amount`, `--category`, `--date`, `--description`
- Reuses `FilterCommand.parseFlags()` to extract update fields
- Merges updated values with existing transaction fields — only provided fields are changed
- Rebuilds the `Transaction` object and calls `setHashId()` to preserve the original ID
- Updates both the `ArrayList` and `HashMap` atomically via `TransactionManager.updateTransaction()`
- Displays a before/after comparison after a successful update

**Command format:** `modify a7b2c3 --amount 25.00 --category dining`

---

#### 3. `SummarizeCommand` (Section 4.6)
Provides a financial overview of transactions with optional filtering.

- Reuses `FilterCommand.buildPredicate()` to support the same filters as `list` (type, category, amount, date range)
- Uses `BigDecimal` for all aggregation to avoid floating-point display errors
- Calculates Total Credit, Total Debit, and Net Balance
- Groups transactions by category and shows sub-totals per category
- Handles empty state gracefully when no transactions match the filter

**Command format:** `summarize`, `summarize --type debit`, `summarize --date-from 2026-01-01 --date-to 2026-03-31`

---

#### 4. `SearchCommand`
Finds transactions by keyword across all searchable fields.

- Searches description, category, HashID, and amount (formatted as `$x.xx`)
- Case-insensitive matching using `.toLowerCase().contains()`
- Reuses `FilterCommand.parseFlags()` to extract the `--keyword` value
- Also registered as `find` (alias) in the Parser
- Displays results in a formatted table identical in style to `ListCommand`

**Command format:** `search --keyword lunch`, `find --keyword food`
